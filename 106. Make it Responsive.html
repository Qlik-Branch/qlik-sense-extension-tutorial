106. Make it Responsive

Learning goal: Define responsive functionality in our extension
Chapter 106 complete code

Qlik Sense out of the box charts are responsive, so we want our extensions to be responsive as well. They should adjust themselves to fit whatever container they are asked to fill in a way that is useful and usable for the user. There are a couple of different ways we can modify our extension based on its size. Let’s review.

Adding Responsiveness via CSS

In CSS, we can define style properties based on the current size of our browser. We do this with what are called “media queries”. Media queries can limit the scope of styles based on features like width and height. We can use media queries to determine if a user is on a phone-size device and apply different styles accordingly.

Let’s add a responsive style to our extension. When viewing our table on a phone, we want the text to be slightly bigger so it’s easier to read. Adding Responsiveness via JS

CSS media queries can help us restyle our extension in response to various browser and device sizes. However, CSS will only get us so far. What if we want to make incremental changes to our visualization on resize, instead of re-rendering the whole thing from scratch with our paint function? That’s where our resize function comes into play.

Recall our extension lifecycle from before. There are two types of events that trigger rendering: data or properties changing, and the chart container changing size.

When only defining a paint function, Qlik Sense will use that function for both types of events. But in our extension, we do not need to re-render the table from scratch on resize. Thanks to Qlik’s CSS, our table does not need any changes in order to resize itself. It will continue to fit into its container. Therefore, running a paint function every time the chart is resized is a waste of time and resources. We can improve the performance of our extension by defining an empty resize function to prevent a re-paint.
